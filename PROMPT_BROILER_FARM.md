# برومت تفصيلي: تطبيق إدارة مزرعة دجاج اللحم (Broiler Farm)

> هذا البرومت يصف تطبيق ويب (PWA) كامل لإدارة مزرعة دجاج اللحم، مبني على نفس فلسفة تطبيق `deku` لمزرعة البيض، لكنه مُكيَّف لدورات تربية الدجاج اللاحم. **يُركّز هذا البرومت على نظام الشركاء وخصائصه بشكل تفصيلي وموسّع.**

---

## 1) نظرة عامة على المشروع

ابنِ تطبيق ويب من صفحة واحدة (Single Page Application) باللغة العربية (RTL) لإدارة مزارع تربية الدجاج اللاحم (Broiler / دجاج اللحم). يجب أن يدعم التطبيق:

- **عدة مصانع/مزارع** لكل صاحب عمل (Multi-factory).
- **عدة دورات تربية متتالية** داخل كل مصنع (Multi-cycle batches).
- **ثلاثة أدوار للمستخدمين**: صاحب عمل (Owner)، شريك (Partner)، عامل (Worker).
- **مزامنة سحابية** لحظية عبر Firebase (Firestore + Auth).
- **عمل دون اتصال** (Offline-first PWA) مع Service Worker و manifest.
- **قابلية التثبيت** كتطبيق على الهاتف (Add to Home Screen).

### التقنيات المطلوبة
- HTML5 + CSS3 خالص (بدون أُطر) + JavaScript (Vanilla ES2020+).
- Firebase Compat SDK 10.x: Auth + Firestore.
- localStorage كطبقة تخزين محلي + مزامنة ذكية مع Firestore.
- خط Cairo من Google Fonts.
- اتجاه `dir="rtl"` و `lang="ar"`.
- أيقونات Emoji بدلاً من مكتبات الأيقونات.
- ثيم داكن احترافي بألوان: ذهبي `#d4a017`، أخضر، أزرق، أحمر، برتقالي، بنفسجي.

## 2) المفاهيم الجوهرية للدجاج اللاحم (مهم جداً للمنطق)

على عكس مزرعة البيض التي تعمل بإدخال يومي مستمر، **مزرعة الدجاج اللاحم تعمل بنظام الدورات (Cycles / Bandes)**. كل دورة تستمر بين 40 و 55 يوماً:

| المرحلة | المدة التقريبية | نوع العلف | ملاحظات |
|---|---|---|---|
| استقبال الكتاكيت | يوم 1 | — | تدفئة، ماء بسكر |
| ستارتر (Starter) | 1-14 يوم | علف 1 (نسبة بروتين عالية) | نمو سريع |
| غرور (Grower) | 15-28 يوم | علف 2 (نسبة بروتين متوسطة) | بناء عضلات |
| فيناشير (Finisher) | 29 - الذبح | علف 3 (نسبة طاقة عالية) | تسمين نهائي |
| الذبح / البيع | 40-55 يوم | — | بيع بالكيلو |

### المؤشرات الأساسية التي يجب حسابها لكل دورة:
- **معدل التحويل الغذائي (FCR — Feed Conversion Ratio)** = إجمالي العلف المستهلك (كغ) ÷ إجمالي الوزن الحي المنتج (كغ). كلما اقترب من 1.6 كان الأداء ممتازاً، وأكثر من 2.0 ضعيف.
- **نسبة النفوق (Mortality %)** = (عدد الكتاكيت النافقة ÷ عدد الكتاكيت الابتدائي) × 100. الطبيعي 3-5%.
- **الوزن الحي المتوسط عند الذبح** (كغ).
- **عمر الذبح** (يوم).
- **مؤشر الأداء الأوروبي (EPEF — European Production Efficiency Factor)** = (نسبة البقاء × متوسط الوزن بالكغ × 100) ÷ (عمر الذبح × FCR). جيد > 300، ممتاز > 400.

## 3) بنية البيانات (Data Models)

### 3.1 المصنع (Farm / Factory)
```js
{
  id: string,
  name: string,
  icon: string,                  // 🐔 🏭 🌾 ...
  ownerUid: string,              // UID صاحب العمل الفعلي
  partnerUids: string[],         // UIDs الشركاء المربوطين
  partnerShares: { [uid]: number }, // نسبة كل شريك
  capacity: number,              // الطاقة الاستيعابية (عدد الكتاكيت)
  hallsCount: number,            // عدد القاعات
  createdAt: timestamp,
  color: string                  // gold | green | blue | red | purple
}
```

## 4) واجهة المستخدم والصفحات الرئيسية

### 4.1 شاشة تسجيل الدخول
- تبويبان: **دخول** / **حساب جديد**.
- في "حساب جديد": اختيار الدور (صاحب عمل أو شريك بدعوة).
- رمز مطوّر اختياري لإنشاء حسابات شركاء محمية.
- الشريك يُسجّل بدعوة من صاحب العمل (دعوة بالبريد).

### 4.2 شاشة اختيار المصنع/المزرعة
- بطاقات لكل مزرعة (مزارعي + المزارع التي أنا فيها شريك).
- إحصائية سريعة: الدورة الحالية، عدد الأيام، نسبة النفوق.
- زر "إضافة مزرعة جديدة" + اختيار أيقونة.

### 4.3 لوحة التحكم (Dashboard)
بطاقات KPI تعرض:
- 🐥 عدد الكتاكيت الحالية في الدورة النشطة
- 💀 نسبة النفوق
- 🌾 إجمالي العلف المستهلك في الدورة
- 📊 FCR الحالي (متوقع)
- ⏱ يوم الدورة (مثلاً: اليوم 23 من 45)
- 💰 المدخول المتوقع للدورة
- 💹 الفائدة الإجمالية الصافية
- 🔮 الفائدة المتوقعة للدورة الحالية
- 💩 فائدة الغبار

### 4.4 صفحة الدورات (Cycles) — جوهر التطبيق
- قائمة كل الدورات (نشطة، منتهية، ملغاة).
- زر "بدء دورة جديدة" يفتح نموذجاً لإدخال:
  - عدد الكتاكيت، السعر، المورّد، السلالة، تاريخ الاستقبال، عدد القاعات المستعملة.
- الدخول إلى دورة يفتح:
  - تبويب "البيانات اليومية" لإدخال النفوق والعلف.
  - تبويب "العلف" لتوزيع كميات الستارتر/غرور/فيناشير.
  - تبويب "الذبح والبيع" (يُفعَّل بعد اليوم 40).
  - تبويب "المصاريف" للأدوية، الكهرباء، الماء، التدفئة.
  - تبويب "تقرير الدورة" مع كل المؤشرات المحسوبة.

### 4.5 صفحة الإدخال اليومي
نفس فكرة تطبيق البيض لكن مُكيَّفة:
- اختيار الدورة النشطة.
- تاريخ، نفوق اليوم، علف اليوم (مع نوعه)، عينة وزن (إن وُجدت).
- مصاريف اليوم (ماء، أدوية).
- سلفيات العمال لذلك اليوم.
- مصاريف الشركاء لذلك اليوم (انظر قسم الشركاء).

### 4.6 صفحة المبيعات والذبح
- إدخال دفعات الذبح (قد تتم على أيام متعددة).
- لكل دفعة: عدد الدجاج، الوزن الحي الكلي، سعر الكيلو، المشتري.
- عرض الباقي في المزرعة وجدول مبيعات تراكمي.
- سجل الكريديات (الديون على الزبائن).

### 4.7 صفحة العمال والشركاء
تبويبان: **العمال** / **الشركاء**.
- العمال: إضافة عامل، نوع الراتب (شهري/لكل دورة/لكل كغ منتج)، علم "عامل الغبار"، علم "عامل التنظيف".
- الشركاء: عرض موسّع — انظر القسم 5.

### 4.8 صفحة التقارير
- ملخص الدورة الحالية.
- مقارنة بين كل الدورات المنتهية (FCR، EPEF، الربح).
- جدول حركة العلف.
- **خانة فائدة الغبار المنفصلة** (مثل تطبيق البيض).
- **خلاصة أرباح الشركاء** (انظر القسم 5).

### 4.9 صفحة الإعدادات
- إعدادات المصنع (الاسم، الطاقة، عدد القاعات).
- التكاليف الثابتة: الكراء، الكهرباء الشهرية، ريباراسيون.
- نسبة صاحب العمل + إدارة الشركاء.
- إنشاء حسابات عمال.
- كلمة سر الحذف.

## 5) نظام الشركاء — التفصيل الكامل (قلب التطبيق)

> هذا هو القسم الأهم. يجب أن يكون نظام الشركاء قوياً، شفافاً، ومُحاسِبياً دقيقاً.

### 5.1 خصائص الشريك (Partner Object)
```js
{
  id: number,
  name: string,
  email: string,                        // للربط بحساب
  uid: string | null,                   // UID بعد قبول الدعوة
  phone: string,
  
  sharePercent: number,                 // نسبة المشاركة الأساسية (%)
  
  // أنواع الحصص المتقدمة
  shareType: 'fixed' | 'capital_based' | 'sliding',
  capitalContribution: number,          // المساهمة الرأسمالية (دج)
  
  // أوزان حسب نوع الفائدة
  weights: {
    production: number,                 // وزنه في فائدة الإنتاج
    manure: number,                     // وزنه في فائدة الغبار
    expansion: number                   // وزنه في فائدة التوسعات/الأصول
  },
  
  // الأذونات
  permissions: {
    canViewAllCycles: boolean,
    canEditDailyEntries: boolean,
    canApproveLargeExpenses: boolean,   // مصاريف فوق حد معين
    largeExpenseThreshold: number,
    canInviteOtherPartners: boolean,
    canViewWorkerSalaries: boolean,
    canExportReports: boolean
  },
  
  // الحالة المالية
  status: 'active' | 'suspended' | 'exited',
  joinDate: string,
  exitDate: string | null,
  exitSettlement: number,               // مبلغ التسوية عند الخروج
  
  // إعدادات السحوبات
  monthlyAllowanceCap: number,          // حد سحب شهري (دج)
  requiresApprovalAbove: number,        // مبلغ يستلزم موافقة
  
  notes: string                         // ملاحظات صاحب العمل
}
```

### 5.2 أنواع الحصص (Share Models)

يجب دعم **ثلاثة نماذج** لتقسيم الفائدة، يختار صاحب العمل واحداً لكل مصنع:

#### النموذج الأول: نسبة ثابتة (Fixed Percentage)
- كل شريك له نسبة ثابتة من الفائدة الصافية.
- مجموع نسب الشركاء + نسبة صاحب العمل = 100%.
- مثال: صاحب العمل 60%، شريك أ 25%، شريك ب 15%.

#### النموذج الثاني: حسب رأس المال (Capital-Based)
- كل شريك يدخل بمبلغ معين كرأس مال.
- النسبة تُحسب تلقائياً = (رأس مال الشريك ÷ مجموع رؤوس الأموال) × 100.
- يجب عرض سجل دخول رأس المال (يدخل تدريجياً عبر "ضخّات" متعددة).

#### النموذج الثالث: نسب متغيرة بحسب نوع الفائدة (Per-Stream Weights)
- كل شريك له **3 أوزان** مختلفة:
  - وزنه في فائدة الإنتاج (لحم).
  - وزنه في فائدة الغبار.
  - وزنه في فائدة الأصول/التوسعات.
- مثال: شريك دخل برأس مال للتوسعة فقط → وزنه 40% في الأصول، 0% في الإنتاج.

### 5.3 المصاريف الشخصية للشريك (Partner Expenses / Withdrawals)

كل شريك يمكن أن يأخذ مبالغ من المزرعة تُسجَّل عليه:

- **الفئات المتاحة**:
  - 💵 سلفية شخصية (مقابل حصة قادمة)
  - 🛒 شراء أدوات للمزرعة (يُحوَّل من حصته إلى نفقات المزرعة)
  - 🍽 ضيافة وعلاقات عامة
  - ⛽ وقود ومواصلات
  - 🏥 طوارئ صحية للدجاج (دفع شخصي يُعوَّض)
  - 🔧 ريباراسيون (يُعوَّض من رصيد المزرعة)

- لكل مصروف: التاريخ، المبلغ، الفئة، الملاحظة، صورة الإيصال (اختياري).
- تُخصم تلقائياً من حصته في الفائدة.

### 5.4 سحوبات الشريك (Partner Drawings)
- يجب التفريق بين **مصاريف للمزرعة** و **سحوبات شخصية**.
- السحب الشخصي لا يحتسب كمصروف للمزرعة، فقط ينقص من رصيد الشريك.
- إنذار عند تجاوز `monthlyAllowanceCap`.
- إنذار "يحتاج موافقة" عند تجاوز `requiresApprovalAbove`.

### 5.5 الفوائد المعروضة لكل شريك
في صفحة التقارير، جدول الشركاء يعرض **سبعة أعمدة لكل شريك**:

| العمود | الحساب | الوصف |
|---|---|---|
| الشريك | — | اسم + شارة الحالة (متصل/في الانتظار/خارج) |
| النسبة % | sharePercent | حسب نوع الحصة |
| المساهمة 🏦 | capitalContribution | مجموع مساهماته الرأسمالية |
| الحصة من الإنتاج 💰 | totalDailyProfit × % | حصته الخام قبل المصاريف |
| الفائدة الصافية 💹 | netProfitBase × % | بعد خصم الكراء، الكهرباء، الكتاكيت، العلف الابتدائي |
| الفائدة المتوقعة 🔮 | expectedCycleProfit × % | بناءً على الأداء الحالي للدورة |
| المصاريف المسحوبة 💸 | sum(partnerExpenses) | مجموع مسحوباته |
| الصافي المستحق 🔥 | الصافي − المسحوبات | المبلغ الفعلي المستحق له |

### 5.6 لوحة الشريك (Partner Dashboard) — وضع الشريك
عند دخول الشريك إلى التطبيق، يرى **واجهة مختلفة** مُخصّصة:
- لوحة تحكم تعرض **حصته فقط** (بدون أرقام الشركاء الآخرين).
- زر "💰 طلب سحب" يفتح نموذج طلب يُرسَل لصاحب العمل للموافقة.
- سجل سحوباته الكامل + رصيده الحالي.
- إذا فعّل صاحب العمل صلاحية "canViewAllCycles": يرى كل الدورات (للقراءة فقط).
- لا يرى رواتب العمال (إلا إذا فعّل `canViewWorkerSalaries`).
- زر "📤 تصدير تقريري" (إذا فعّل `canExportReports`) لتصدير تقرير PDF/Excel بحصته.

### 5.7 آلية الدعوة (Partner Invitation Flow)
1. صاحب العمل يدخل اسم الشريك + بريده + نسبته → يُحفظ كـ "في الانتظار ⏳".
2. النظام ينشئ سجل دعوة في Firestore: `invitations/{email}`.
3. عند تسجيل الشريك بنفس البريد، يُكتشف السجل ويُربط الـ UID تلقائياً.
4. تُرسل إشعارات داخل التطبيق لكلا الطرفين.
5. الشارة تتحول من "في الانتظار ⏳" إلى "متصل 🔗".

### 5.8 خروج شريك (Partner Exit)
- زر "إخراج الشريك" يفتح نموذج تسوية:
  - حساب رصيده الحالي + حصته من الأصول التراكمية.
  - عرض مبلغ التسوية المقترح.
  - تأكيد بكلمة سر صاحب العمل.
- بعد الخروج، يبقى السجل التاريخي للقراءة فقط.
- نسبته تُعاد توزيعها يدوياً بين الباقين أو لصاحب العمل.

### 5.9 سجل التغييرات (Partner Audit Log)
كل تعديل على شريك (تغيير نسبة، إضافة مصروف، تعديل صلاحية) يُسجَّل في:
```js
{
  partnerId, action, by_uid, before, after, timestamp, note
}
```
- الشريك يستطيع رؤية سجل تعديلاته.
- يُعرض كـ "Activity Feed" زمني داخل بطاقة الشريك.

### 5.10 إنذارات الشركاء (Partner Alerts)
- 🔔 شريك تجاوز سقفه الشهري.
- 🔔 شريك يحتاج موافقة على مصروف كبير.
- 🔔 شريك جديد أكمل التسجيل (للموافقة النهائية).
- 🔔 مجموع نسب الشركاء ≠ 100% (تنبيه تكوين).
- 🔔 رصيد شريك أصبح سالباً (سحب أكثر من حصته).

### 5.11 تقرير شامل لكل شريك (Per-Partner Statement)
زر "📊 كشف حساب" بجانب كل شريك يفتح تقريراً يحتوي:
- معلوماته الأساسية ونوع حصته.
- جدول كل الدورات: حصته فيها، مصاريفه، صافيه.
- رسم بياني (Chart) يُظهر تطور حصته عبر الزمن.
- إجمالي ما ربحه + إجمالي ما سحبه + الرصيد الحالي.
- زر تصدير PDF بختم المزرعة.

### 5.12 المقارنة بين الشركاء
في صفحة التقارير، قسم "📊 مقارنة الشركاء":
- جدول يقارن كل الشركاء على نفس المؤشرات.
- رسم بياني عمودي للحصص والمسحوبات.
- "نسبة الكفاءة" = (الصافي المستحق ÷ المساهمة الرأسمالية) × 100 — مؤشر العائد على الاستثمار لكل شريك.

## 6) منطق الحسابات (Calculations)

### 6.1 الفائدة الأساسية للدورة (Base Profit)
```
revenue        = totalLiveWeight × pricePerKg + manureIncome
costs          = (initialChicks × chickPrice)
                + (starterUsed × starterPrice)
                + (growerUsed × growerPrice)
                + (finisherUsed × finisherPrice)
                + medicationsCost + vaccinesCost
                + electricityCost + waterCost
                + heatingCost + beddingCost
                + (workerAdvances NOT for manure worker)

baseProfit     = revenue - costs
```

### 6.2 الفائدة الصافية الإجمالية
```
netProfit = baseProfit
            - effectiveLoyer (الكراء - ريباراسيون الكراء)
            - totalElectricityFixed (الكهرباء الشهرية × عدد الأشهر)
            - repairTotal (ريباراسيون الفائدة)
            - totalCredits (الكريديات/الديون المستحقة)
            - totalPartnerExpenses
            - totalOwnerAdvances
```

### 6.3 الفائدة المتوقعة للدورة الحالية
```
elapsedDays    = today - cycle.startDate
expectedDays   = 45 (افتراضي)
progress       = elapsedDays / expectedDays

projectedRevenue = (currentLiveWeight × expectedFinalWeight/currentAvgWeight) × pricePerKg
projectedFCR     = currentFCR (يتم تحديثه)
expectedProfit   = projectedRevenue - projectedTotalCosts
```

### 6.4 فائدة الغبار (Manure Profit)
```
totalManureRevenue   = sum(cycle.manureIncome for all cycles)
manureWorkerAdvances = sum(advances of worker where isManureWorker == true)
netManureProfit      = totalManureRevenue - manureWorkerAdvances
```

### 6.5 حصة الشريك (Partner Share)
```
// النموذج البسيط
partnerShare = netProfitBase × (sharePercent / 100)

// النموذج بالأوزان
partnerProductionShare = productionProfit × (weights.production / 100)
partnerManureShare     = netManureProfit  × (weights.manure / 100)
partnerExpansionShare  = expansionProfit  × (weights.expansion / 100)
partnerTotalShare      = sum(all three)

// الصافي المستحق
partnerNetDue = partnerTotalShare - sum(partnerExpenses)
```

## 7) الأمان والصلاحيات

- صاحب العمل: صلاحية كاملة على مصنعه.
- الشريك: قراءة فقط افتراضياً + ما تم تفعيله في `permissions`.
- العامل: إدخال يومي فقط، لا يرى المصاريف الكبرى ولا أرباح الشركاء.
- كل تعديل حساس (حذف دورة، تعديل نسبة شريك، إخراج شريك) يستلزم كلمة سر إضافية.
- قواعد Firestore Security Rules تمنع:
  - شريكاً من رؤية بيانات مصنع آخر.
  - عاملاً من تعديل الإعدادات أو الشركاء.
  - أي مستخدم من تعديل سجل تغييرات شريك آخر.

## 9) ميزات إضافية مهمة

- **اقتراح دورة جديدة**: زر يُحلّل آخر دورة منتهية ويقترح: عدد الكتاكيت، الموسم الأمثل، توقع الربح.
- **تنبيه FCR سيئ**: إذا تجاوز 2.0 منتصف الدورة، تنبيه أحمر مع توصيات.
- **تنبيه نفوق غير طبيعي**: إذا تجاوز نفوق يوم > 1% من القطيع.
- **حاسبة العلف**: تقدير الاحتياج المتبقي حسب يوم الدورة وعدد الدجاج الحالي.
- **مقارنة الموردين**: تخزين تقييمات لكل مورّد كتاكيت/علف.
- **تنبيه الذبح**: إشعار عند بلوغ متوسط الوزن المستهدف.
